## 1. Введение

**Объём**: 2–3 страницы.

**Обязательные элементы (Прил. 1 Положения)** — каждый присутствует явным абзацем или подзаголовком. JUDGE_VKR Phase 2 проверяет каждый из них как `PASS`/`FAIL`.

### 1.1. Обоснование выбора темы и актуальность
- Обратная задача мультипольного анализа: восстановление коэффициентов $(a^E, a^M)$ по диаграмме направленности — ключевой шаг в характеризации антенн и излучающих структур.
- Практическая постановка: измеряется только мощностная диаграмма $P(\theta,\varphi) = |E_\theta|^2 + |E_\varphi|^2$, фаза $E_\theta, E_\varphi$ недоступна; классическое ортогональное разложение через $\mathbf B \cdot \mathrm{vec}(E_{UT})$ не применимо.
- Задача структурно некорректна по Адамару: глобальная фаза $U(1)$ + reflected-conjugate symmetry (Bangun 2020) → $\mathcal A^{-1}(P_{UT})$ содержит как минимум два топологически различных компонента.
- Существующие решения: коммерческие симуляторы (HFSS, CST), специализированные алгоритмы phase retrieval. Применение методов машинного обучения к данной комбинации (безфазовое измерение + сферическая геометрия + полные векторные сферические гармоники) — открытое направление, которое не имеет общедоступного бенчмарка.

### 1.2. Объект и предмет исследования
- **Объект**: отображение $P_{UT} \rightarrow (a^E, a^M)$ как обратная задача восстановления мультипольных коэффициентов из мощностной диаграммы направленности.
- **Предмет**: машинно-обученный регуляризатор как способ приближения одного физически разумного прообраза из множества $\mathcal A^{-1}(P_{UT})$.

### 1.3. Цели работы

**Главная цель**: исследовать возможность применения методов машинного обучения для решения обратной задачи мультипольного анализа в условиях безфазового измерения и построить воспроизводимый бенчмарк для последующих исследований.

**Операционные цели** (с измеримыми критериями успеха):
1. Сформулировать обратную задачу формально и выявить её математический характер (некорректность, симметрии).
2. Построить пайплайн генерации обучающих данных: синтетический + реальный с аугментацией.
3. Придумать и протестировать набор моделей и функций потерь; сравнить их на единых метриках.
4. Провести экспериментальное исследование: вклад модели, признаков, потерь, режима генерации, аугментаций.
5. Оформить программный фреймворк как воспроизводимую кодовую базу для дальнейших исследований.

### 1.5. Перечень методов исследования

Перечень ограничен методами и инструментами, явно зафиксированными в [paper/proposal.md](proposal.md) (см. принцип № 3).

- **Аналитические методы**: теория электромагнитного излучения и VSH (Jackson 1999, Hansen 1988); теория некорректных обратных задач по Адамару (Arridge et al. 2019); теория phase retrieval и связанных симметрий (Shechtman 2015, Fienup 1982, Bangun 2020).
- **Численные методы**: рандомизированный SVD для PCA-снижения размерности (Halko-Martinsson-Tropp 2011), 2D FFT для радиального спектра, скалярная SH-декомпозиция для спектральной мощности.
- **Методы машинного обучения** (по [paper/proposal.md](proposal.md)): feed-forward MLP-backbone (5×200); многоголовая декомпозиция с Матрёшка-trick — обучение голов от низких порядков коэффициентов к высоким, с разными политиками для backbone (заморозка после первого этапа, заморозка только предыдущих голов, увеличение lr на текущей голове); дифференцируемый VSH-декодер для прокидывания градиентов из пространства поля в пространство коэффициентов.
- **Эксперимент** (по [paper/proposal.md](proposal.md)): пошаговое исследование вклада рычагов — base → регуляризация (функции потерь) → признаки → расписание обучения голов; измерение метрик качества на трёх типах наборов оценки — идеальном (реальная антенна или симуляция с истинными коэффициентами), holdout (часть реальной выборки, не использованная в аугментациях) и dummy (рандомно сгенерированные коэффициенты с известными полями).

### 1.6. Научно-теоретическая и практическая значимость

- **Теоретическая**: формализация обратной задачи мультипольного анализа как задачи безфазового восстановления коэффициентов VSH; явная классификация тривиальных симметрий ($U(1)$ + reflected-conjugate, по Bangun 2020).
- **Практическая**: воспроизводимый бенчмарк (фреймворк `mpinv`) с открытыми датасетами, конфигурациями и результатами; рабочий ML-baseline для обратной задачи $P_{UT} \rightarrow (a^E, a^M)$ на сферической сетке для полного набора векторных сферических гармоник; рекомендации по выбору функции потерь и режима генерации синтетической выборки.

### Глава 1. Физическая постановка задачи мультипольного анализа

**Бюджет**: 5–7 страниц.

**Источник**: перевод [presentation/ch1_full.md](../presentation/ch1_full.md) §§ 1.1–1.8 на русский. Цитаты — Jackson 1999, Hansen 1988, Shechtman 2015, Fienup 1982, Bangun 2020, Arridge 2019; backed by [research/ch1-enrichment/manifest.md](../research/ch1-enrichment/manifest.md).

**Параграфы**:
- 1.1. Мультипольное разложение в электромагнетизме: векторные сферические гармоники и коэффициенты. Базис $\boldsymbol\Psi^E_{lm}, \boldsymbol\Psi^M_{lm}$, упаковка $[\mathrm{Re}\,a^E, \mathrm{Im}\,a^E, \mathrm{Re}\,a^M, \mathrm{Im}\,a^M]$, фиксация $L = 5$, $K = 35$, размер вектора $4K = 140$.
- 1.2. Диаграмма направленности как объект анализа: комплексное поле $E_{UT}$ на сферической сетке 360 × 179, исключение полюсов, $\dim_{\mathbb R}(E_{UT}) = 257{,}760$.
- 1.3. Аналитический метод разложения: скалярные произведения с базисными модами; матрица $\mathbf B \in \mathbb C^{70 \times 128880}$; $\hat a = \mathbf B \cdot \mathrm{vec}(E_{UT})$ как gold standard для полностью измеренного комплексного случая.
- 1.4. Практическая проблема: измерение мощности $P = |E_\theta|^2 + |E_\varphi|^2$ вместо комплексного поля; четырёхкратный коллапс информации до $64{,}440$ вещественных чисел на грид.
- 1.5. Формальная постановка обратной задачи: $P_{UT} \rightarrow (a^E, a^M)$; разложение forward-оператора $\mathcal A = |\cdot|^2 \circ \mathcal S$; синтез $\mathcal S$ линеен, $|\cdot|^2$ нелинеен и many-to-one.
- 1.6. Информационный коллапс при переходе $E \to P$: per-sample 4-в-1 в смысле $S^3$-преобраза; глобальный учёт степеней свободы; две концептуальные неоднозначности.
- 1.7. Строгая некорректность: классификация Адамара; $U(1)$ + reflected-conjugate (Bangun 2020 Ch. 6 §6.2).
- 1.8. Машинное обучение как регуляризация: вариационный шаблон $\arg\min(\|\mathcal A(a) - P\|_w^2 + \lambda R(a))$; почему классические Tikhonov/sparsity/TV-регуляризаторы не подходят; обоснование, что прямая регуляризация заменяется на обученный prior, имплицитно заданный обучающей выборкой.

**Зависимости от других глав**: forward-references в Главу 2 (данные/аугментации), Главы 3–4 (архитектуры/потери), Главу 6 (эксперименты).

**Замечания**: текущий драфт перегружен относительно бакалаврского бюджета; при переводе сжать §§ 1.6–1.7 (объединить «информационный коллапс» с «строгой некорректностью» в один параграф), удалить дубли с §§ 1.3–1.4.

### Глава 3. Архитектуры моделей

**Бюджет**: 3–4 страницы.

**Источник**: новая глава, опирается строго на [paper/proposal.md](proposal.md) (раздел «Архитектура нейронной сети»). Программные артефакты: [src/mpinv/models/mlp.py](../src/mpinv/models/mlp.py), [src/mpinv/models/multi_head_mlp.py](../src/mpinv/models/multi_head_mlp.py), [src/mpinv/losses/differentiable_field.py](../src/mpinv/losses/differentiable_field.py). Линейный baseline в работу **не входит** (см. принцип № 6).

**Параграфы**:

- **3.1. MLP-backbone.** Архитектура `flat`; ширина 5×200 по `proposal.md`. SiLU, AdamW. MLP выбран как базовая модель с достаточной мощностью и без избыточных индуктивных предпосылок: входные признаки уже декоррелированы PCA или составным CV-стеком (Глава 2), и роль модели — выучить регулярность связи $\text{features} \to (a^E, a^M)$ по обучающей выборке, не привнося дополнительных предположений.

- **3.2. Многоголовая модель и Матрёшка-trick.** `MultiHeadMLP` с общим backbone и отдельными головами на каждый порядок $l$. Идея напрямую из [paper/proposal.md](proposal.md): «**Поскольку коэффициенты первых порядков имеют большую амплитуду в наших данных и на практике, имеет смысл учить головы сначала для первых порядков коэффициентов, и только потом для более высоких.**» Поверх модели, обученной на $L = 5$, можно обучать модели более высоких порядков благодаря тому же приёму.

  **Способы обучения голов** (из `proposal.md`):
  - заморозка backbone-модели после первого этапа;
  - заморозка только предыдущих голов, backbone остаётся обучаемым;
  - увеличение learning rate на текущей голове без заморозки предыдущих и backbone.

  **Способы вычисления потери на голове $k$** (из `proposal.md`):
  - по всему $P$ (поле, восстановленное по всем коэффициентам всех активных голов);
  - по $P$, полученному только по коэффициентам до головы $k$ включительно.

  Конкретный выбор схемы — предмет экспериментального сравнения (Глава 6).

- **3.3. Дифференцируемый VSH-декодер.** Из [paper/proposal.md](proposal.md): «Берёт output модели (коэффициенты) и получает по ним поле $\hat P$. Через VSH-декодер текут градиенты для обучения.» Реализация — [src/mpinv/losses/differentiable_field.py](../src/mpinv/losses/differentiable_field.py) (`DifferentiableMultipoleField`): precomputed комплексный VSH-базис, einsum-синтез поля, $|\cdot|^2$ для $P$. Идея таких декодеров не нова в literature; для данной задачи он реализован специально и используется только во время обучения с потерей в пространстве поля; на инференсе декодер не вызывается.

**Что НЕ входит в эту главу**: гиперпараметры обучения, схедулеры, оптимизаторы — Глава 4. Конкретные конфигурации, выигрывающие в экспериментах — Глава 6. Описание реестра `MODELS` — Глава 7.

### Глава 4. Функции потерь и обучение

**Бюджет**: 3–4 страницы.

**Источник**: новая глава, опирается на [paper/proposal.md](proposal.md) (раздел «Функции потерь» и «Обучение»). Программные артефакты: [src/mpinv/losses/](../src/mpinv/losses/), [src/mpinv/training/](../src/mpinv/training/), конфиги в `configs/loss/`.

**Параграфы**:

- **4.1. Прямая регрессия на коэффициенты `CoefMSE`.** MSE между предсказанным и истинным packed-вектором $a$. Формула, конвенция «no normalisation on target». Чувствительность к амплитудам мод низких $l$.

- **4.2. Физически-информированная функция потерь `PhysicsPowerLoss`.** Декодирование предсказанных коэффициентов в $\hat P$ через `DifferentiableMultipoleField`; вычисление $\|\hat P - P\|_w^2$ с весом $\mu(\theta) = \sin\theta\,\Delta\theta\,\Delta\varphi$. Опциональный `log_ratio` режим; параметр `coef_aux_weight` (mixed loss `physics_power_mixed`).

- **4.3. Ранжирующая потеря `RankBinPLoss` на $P$.** **Очень важный пункт** ([paper/proposal.md](proposal.md), раздел «Функции потерь»): «**Добавление этого loss''а позволяет модели сходиться. Без него модель не лучше рандома и запоминания шума.**» Soft quantile binning через сигмоиду; default `n_bins = 2 \cdot l_{\max} + 1`; используется как самостоятельный лосс или как `rank_bin_weight` внутри `physics_power_rank`. Мотивация — из принципа № 7: ранжирующая потеря на $P$ — собственный приём этого исследования и центральный элемент сходимости моделей.

- **4.4. Композитные потери.** `physics_power_mixed` (физика + auxiliary coef MSE с весом 0.1); `physics_power_rank` (физика + ранжирующий компонент с весом 0.1) — лучший по результатам экспериментов с реальными данными. Какой компонент за что отвечает.

- **4.5. Тренировочный цикл.** Кастомный `Trainer` в [src/mpinv/training/trainer.py](../src/mpinv/training/trainer.py); callback-driven обучение (`CheckpointCallback`, `EarlyStoppingCallback`, `GradClipCallback`, `LoggingCallback`, `TimingCallback`, `ValidationCallback`); LR scheduler per-step.

- **4.6. StagedTrainer для Матрёшки.** Per-$l$-band обучение; head lifecycle (zero-init future heads, reinit active head); `transplant_heads` при переходе $L = 5 \to L = 15$; backbone policies (см. § 3.2).

- **4.7. Гиперпараметры.** Оптимизатор: AdamW (default) или SGD+Nesterov с cosine warmup (для производственной конфигурации эксперимента с реальными данными); learning rate (как обоснован выбор); количество эпох (с ранней остановкой `patience = 10`–20); batch size; gradient clipping `1.0`. Конкретные значения для каждой экспериментальной серии — в Главе 6.

**Что НЕ входит**: метрики качества — Глава 5; экспериментальные результаты — Глава 6.

## 9. Открытые риски

### 9.1. Volodya `MAJOR REVISION` по Главе 2
Текущий английский драфт [presentation/paper_full.md](../presentation/paper_full.md) §§ 2.1–2.6 имеет открытые дефекты ([presentation/ch2_review.md](../presentation/ch2_review.md), [presentation/ch2_review_1.md](../presentation/ch2_review_1.md)):

- Notation collisions: $p$ ↔ Bernoulli vs PCA-oversampling; $r$ ↔ PCA-rank vs residual fraction; $X$ ↔ family index vs data matrix vs Fourier transform; $a$ ↔ packed vector vs antenna radius.
- Неопределённые $n_\mathrm{src}, n_\mathrm{tr}, n_\mathrm{ho}, N_\mathrm{aug}$ при заявлении «training distribution itself is fully specified» в опенере.
- «Well-known pole-region bias» без цитаты.
- Заявление «can ever recover» — overreach.
- snake_case-идентификаторы в augmentation chain без сопоставления с пронумерованными примитивами.

**Стратегия закрытия**: при переводе Главы 2 на русский (см. Таблицу § 8) переименовать конфликтующие символы (`p_oversampling = q`, `residual_fractions = ρ_P, ρ_E`, `data matrix = Φ`, `antenna radius` — изъять); зафиксировать $n_\mathrm{src}, n_\mathrm{tr}, n_\mathrm{ho}, N_\mathrm{aug}$ через явную отсылку в Главу 6, где значения проставлены в таблицах; добавить цитату Hansen 1988 для pole-region bias или ослабить формулировку; ослабить «can ever recover» до «не воспроизводятся в bandlimited resynthesised поле».

Сохранить в `paper/figures/`.

### 9.3. Расхождение между двумя ведущими конфигурациями эксперимента с реальными данными
- «Лучшая по валидации» конфигурация: меньшая модель, ранжирующая физическая потеря, 100 источников / 1k augmented, 10 эпох; даёт хорошую внутри-источниковую метрику $P$.
- «Производственная» конфигурация: 5×200 MLP, прямая регрессия на коэффициенты, 200 источников / 10k augmented, 30 эпох; более крупная и ближе к целевому масштабу.
